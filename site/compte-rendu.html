Présentation du Projet

VELORESTAU est une application web permettant de visualiser les stations de vélos et les restaurants à Nancy. Cette application a été développée dans le cadre d'une SAE de 2ème année de BUT Informatique.

Architecture du Projet

Technologies Employées

  • Frontend : HTML, CSS, JavaScript
  • Backend : Java
  • Librairies : Lucide Icons, Leaflet.js
  • Services : RMI, HttpServer (Proxy) et serviceDb (RMI)
  • Base de données : Oracle (Charlemagne)
  • Hébergement : Webetu

Schéma de la base de données

Réalisations

Services Implémentés

  • Service base de données (RMI)
    • Inscription au serveur central via RMI
    • Récupération des informations des restaurants depuis la base de données Oracle
    • Envoi des données au serveur central via RMI
  • Service Proxy (HttpServer)
    • Récupération des données des stations vélibs via les données ouvertes
    • Récupération des incidents de circulation
    • Gestion des données bloquées par le proxy IUT
    • Envoi des données au serveur central via RMI
  • Serveur central (RMI + HTTP)
    • Réception des données des services via RMI
    • Exposition d'API REST (/api/restaurants, /api/velibs, etc.)
    • Gestion centralisée des données
    • Interface web pour la visualisation

Aspects Techniques

  • Compilation automatique du projet
  • Documentation Javadoc de l'API publique
  • Gestion des erreurs réseau et des timeouts
  • Configuration du proxy IUT pour l'accès aux données externes

Guide d'Utilisation

Prérequis

  • Java 8 ou supérieur
  • Oracle Database (sur Charlemagne)
  • Un navigateur web moderne

Configuration

  • Configuration du proxy IUT pour l'accès externe (conf.ini)
  • Paramètres de connexion à la base de données Oracle (conf.ini)
  • Configuration des services RMI (conf.ini)

Hébergement

Le site web est hébergé sur Webetu, un serveur de l'IUT.

Les services sont lancés sur différentes machines de la salle.

Le site web est accessible à l'adresse : https://webetu.iutnc.univ-lorraine.fr/www/e0617u/SAE_VELORESTAU/
